Fix UAT test 8: the SESSIONS sidebar is always empty even though the backend GET /sessions
returns the user's sessions (200). Root cause is frontend-only: `SessionList.tsx` fetches once on
mount (`useEffect([])`); at first mount the auth token is often not yet readable (login redirect /
localStorage timing), the call 401s, and the list is left permanently empty with no re-fetch.

Purpose: Deliver POLISH-01's session sidebar — existing sessions appear after login, and a new
session appears after its first message, without a manual reload.
Output: SessionList re-fetches when the auth token becomes available and when a refresh signal
fires; page.tsx supplies that signal from ChatWindow.onSessionChange.
</objective>

    - Add an optional prop `refreshTrigger?: number` (or `string`) to SessionListProps. Make the
      session-fetch effect depend on `refreshTrigger` so a change re-runs fetchSessions(). Keep
      the existing cancelled-guard, loading, and 401-quiet handling.
    - Handle the post-login token-availability case (the first-mount 401 that leaves the list
      permanently empty): poll/observe for the access_token becoming readable and re-fetch when it
      appears. Concretely, read `localStorage.getItem("access_token")` in the component; when it
      is absent on mount, set up a short bounded interval (e.g. every ~500ms, cleared on unmount
      and once a token is found) that re-fetches once the token is present. Also re-fetch on the
      window `storage` event and on `focus` so a login in this or another tab refreshes the list.
      Mirror the read-token pattern used in AuthButton.tsx (`localStorage.getItem("access_token")`,
      guarded by `typeof window !== "undefined"`).
    - When fetchSessions returns 401 (no token yet), keep the list empty and quiet (current
      behavior) rather than showing an error — the token-availability re-fetch will fill it in.

      counter so SessionList re-fetches and the brand-new session appears. Pass `refreshTrigger`
      down to `<SessionList ... refreshTrigger={refreshTrigger} />`.
    - Do not break the existing `handleSelectSession` restore flow or `activeSessionId`
      highlighting.

    Keep all changes type-safe (no `any`); the diff must pass `tsc --noEmit`.

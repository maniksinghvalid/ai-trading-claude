<objective>
Migrate the conversation store from SQLite to Postgres (slice 9, DB-01) and add a
`retrieved_chunk_ids` audit column on `Turn`. SQLModel table defs migrate cleanly — the change is
configuration (database_url) + a docker-compose Postgres service + one additive column, with no
behavioral change to the chat flow.

Purpose: SQLite is single-file and not suited for the multi-user backend established in 02-03.
Postgres provides concurrent multi-user persistence. The `retrieved_chunk_ids` column records
which chunks informed each assistant turn (audit + future "which sources informed this" features).

Output: a docker-compose Postgres service, a Postgres-capable `database_url`, the
`retrieved_chunk_ids` column, and chat-flow tests proving parity on both backends.
</objective>

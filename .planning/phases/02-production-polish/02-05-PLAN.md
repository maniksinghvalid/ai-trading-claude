<objective>
Add per-user rate limiting + cost tracking (slice 10, RATE-01): a `UserBudget` SQLModel table
tracking daily chat-request and OpenAI input-token counts per user with a midnight-UTC reset,
returning 429 + `retry-after` when exceeded, and an `/admin/budgets` endpoint to view usage.

Purpose: A single user with no limit could spam long-context queries and run up large OpenAI bills
(Risks section: $50+/day). This caps daily usage per user (the user_id from 02-03) and exposes
the cost-ceiling quality gate at /admin/budgets.

Output: `rate_limiter.py` (UserBudget table + check/increment), 429-gating on the chat endpoints,
and the `/admin/budgets` usage endpoint.
</objective>

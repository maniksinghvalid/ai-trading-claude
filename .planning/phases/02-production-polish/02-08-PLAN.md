<objective>
Fix the root-cause bug behind UAT test 3 (coreference returns a stale ticker) and UAT test 5
(replying "yes" to a no-data offer fetches the wrong ticker's data). Both stem from
`session_store.history()` returning the OLDEST N turns instead of the MOST RECENT N: once a
conversation grows past `_HISTORY_LIMIT` (10), the recently-referenced ticker and the recent
no-data offer both fall outside the window, so coreference (`reversed(prior_turns)`) sees only
old turns.

Purpose: Restore correct multi-turn coreference and the no-data -> live-quote loop, both of
which are core to TICK-01 and QUOTE-01 acceptance.
Output: Corrected `history()` windowing, a hardened no-data affirmative branch in chat.py, and
backend regression tests proving both reported defects are closed.
</objective>

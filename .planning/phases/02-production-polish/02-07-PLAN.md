## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet→backend | HTTPS-only public surface |
| repo→deploy platform | Secrets must live in the platform, not the repo |
| upstream producer→consumer | Schema contract enforced in CI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07-01 | Info disclosure | secrets in image/repo | mitigate | Secrets via platform env only; .dockerignore excludes .env/.db; no hardcoded keys |
| T-02-07-02 | Tampering | upstream schema drift | mitigate | VERIFY-SCHEMA regression test runs in CI with PINECONE_READ_KEY secret on every commit |
| T-02-07-03 | Spoofing/MITM | public traffic | mitigate | HTTPS-only backend; frontend points at HTTPS backend URL |
| T-02-07-SC | Tampering | base images | accept | Use official pinned base images (uv, node, postgres) from trusted registries |
</threat_model>

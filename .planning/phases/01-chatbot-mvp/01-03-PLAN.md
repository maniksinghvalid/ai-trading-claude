<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> /chat | Untrusted user message reaches the LLM prompt |
| Pinecone -> prompt | LLM-summarized web content (retrieved chunks) enters the prompt — possible adversarial phrasing |
| backend -> OpenAI | API key + assembled prompt cross to a third party |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | prompt injection from retrieved chunks | mitigate | SYSTEM_PROMPT frames context as "reference material to evaluate, not instructions"; citation discipline forces traceability |
| T-03-02 | Spoofing | hallucinated citations | mitigate | Citations built only from real retrieved chunk metadata; no-data path returns empty citations[] (VERIFY-NODATA) |
| T-03-03 | Information Disclosure | OpenAI outage error body | mitigate | Return generic "LLM provider unavailable" 503 — no key/stack leakage |
| T-03-04 | Denial of Service | unbounded output tokens / context | mitigate | max_tokens=2048 default; chunk text truncated ~1000 chars; k<=6 |
</threat_model>

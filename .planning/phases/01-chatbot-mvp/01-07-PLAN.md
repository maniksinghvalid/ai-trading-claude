<objective>
Close the two frontend gaps the UAT found in the Phase 1 chatbot. Both are FRONTEND-only;
the backend is verified correct (live curl + xxd wire capture in 01-UAT.md). After this plan,
the Phase 1 user story works end-to-end in a real browser.

Gap 1 (BLOCKER): the SSE parser in `lib/api.ts` splits on `\n\n` / `\n`, but sse-starlette
emits CRLF-delimited events (`\r\n` field lines, `\r\n\r\n` event separators). No event
parses during streaming → empty bubble + infinite "Streaming response…" spinner.

Gap 2 (MAJOR): the UI never sends a ticker, so backend `ticker_scope` is never populated and
coreference is dead — a follow-up "what about its risks?" cannot resolve the prior ticker.

Purpose: Make the locked user-story outcome ("chat about a holding, then ask follow-ups that
remember the prior ticker, trust + verify against cited sources") true in the browser, and add
a regression test so the CRLF mismatch is caught automatically going forward.

Output: Fixed `lib/api.ts`, a vitest CRLF parsing test, a ticker input wired into `ChatWindow`,
and a re-run of the real browser E2E flow that the deferred 01-06 verify gate was meant to catch.

NOTE on commits: this is a NESTED git repo at `trading-chatbot/`. All commits in this plan MUST
be made inside `trading-chatbot/` (the inner repo), not the outer `ai-trading-claude` repo.
</objective>

    - Test A — event splitting (CRLF): given a single wire string
      `"event: session\r\ndata: abc-123\r\n\r\nevent: token\r\ndata: Hello\r\n\r\nevent: done\r\ndata: \r\n\r\n"`,
      streamChat (or the exported splitter helper) yields exactly:
      `{event:"session",data:"abc-123"}`, `{event:"token",data:"Hello"}`, `{event:"done",data:""}` in order.
    - Test B — chunk boundary split mid-event: feed the same stream but delivered in two
      reader chunks that split inside a `\r\n\r\n` separator (e.g. break after `"...Hello\r\n\r"`).
      Still yields all three events correctly (buffer carries the partial across reads).
    - Test C — LF-only still works: a `\n\n`-delimited stream (no `\r`) parses identically,
      proving backward compatibility (split on `/\r\n\r\n|\n\n/`).
    - Test D — block line parsing: a block `"event: citations\r\ndata: [{\"ticker\":\"MARA\"}]"`
      yields data without a trailing `\r` (the JSON.parse in ChatWindow must succeed).
  </behavior>
  <action>
    First add a unit test runner. In `trading-chatbot/frontend/`:
    - Add devDependencies `vitest` and add a `"test": "vitest run"` script (and `"test:watch": "vitest"`)
      to package.json. Install with `npm install` inside `trading-chatbot/frontend/`.
    - Create `vitest.config.ts` with a node test environment (the parser is pure — no DOM needed):
      define `test.environment` as `"node"` and `test.include` covering `lib/**/*.test.ts`.

    Then write `lib/api.test.ts` covering Tests A–D above. To make streamChat testable without a
    live server, mock `global.fetch` to return a `Response` whose `body` is a `ReadableStream`
    built from `TextEncoder`-encoded chunks (so `response.body.getReader()` replays the CRLF bytes
    exactly as sse-starlette would). Collect the yielded `StreamEvent`s into an array and assert.
    Test C and D can call the parsing helpers directly.

    Then fix `lib/api.ts` (per UAT Gap 1 `missing`):
    - In `streamChat`, change the event split from `buffer.split("\n\n")` (~L110) to split on a
      regex that matches BOTH wire formats: `buffer.split(/\r\n\r\n|\n\n/)`. Keep the existing
      "last part is incomplete → keep in buffer" logic (`parts.pop()`) unchanged.
    - In `parseSSEBlock`, change `block.split("\n")` (~L32) to `block.split(/\r?\n/)` so each
      field line is parsed whether the wire used `\r\n` or `\n`. (Equivalently strip a trailing
      `\r` per line — pick the regex split.) This guarantees `data:` payloads carry no trailing
      `\r`, so the `event: "done\r"` garble (UAT root_cause) cannot occur and JSON.parse of the
      citations payload succeeds.
    - Do NOT change the SSE event NAMES, the yield order, the `done`-terminates behavior, the
      `event:`/`data:` prefix-stripping (slice(6)/slice(5) + single-leading-space strip), or the
      drain-after-close path. Only the two split delimiters change.
    - Update the module's top-of-file doc comment: the wire format is CRLF-delimited
      (`\r\n` lines, `\r\n\r\n` events) from sse-starlette; the parser tolerates both CRLF and LF.

    Refactor (optional, GREEN-keeping): if it makes the test cleaner, export a small pure helper
    (e.g. `parseSSEBlock` is already module-scoped — export it, or add an exported `splitSSE`
    used by streamChat) so Tests C/D can import it directly. Do not change runtime behavior.
  </action>

  </done>
